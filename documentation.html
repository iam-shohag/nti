1. Overview

This eCommerce database is designed for a comprehensive online shopping platform with support for:

  • Multi-product catalog management
  • Hierarchical category structure
  • Shopping cart and wishlist functionality
  • Order processing and management
  • Multiple payment gateways (Stripe, PayPal, bKash, Nagad, Rocket, SSL Commerce)
  • User authentication and authorization
  • Reward points/coins system
  • Coupon and discount management
  • Product reviews and ratings
  • Newsletter subscriptions
  • Bangladesh-specific address system (Districts, Upazilas)

2. Architecture

2.1 Database Engine

Engine: InnoDB

Character Set: utf8mb4

Collation: utf8mb4_unicode_ci

Version: MySQL 8.0.30

2.2 Architecture Layers

Three-Tier Architecture

Presentation Layer: Frontend (Customer & Admin interfaces)

Application Layer: Business Logic (Laravel Framework)

Data Layer: MySQL Database (This schema)

2.3 Design Principles

  • Normalization: Database follows 3NF (Third Normal Form) to minimize redundancy
  • Referential Integrity: Foreign key constraints ensure data consistency
  • Scalability: Indexed columns for optimal query performance
  • Flexibility: JSON fields for extensible data storage
  • Security: Encrypted sensitive data (passwords, API keys)

4. Product Management

Products Table Structure

Key Features:

  • Unique SKU for inventory tracking
  • SEO-friendly slugs for URLs
  • Price and compare_at_price for discounts
  • Stock management
  • Featured products flag
  • Custom page builder support (JSON)
  • Meta fields for SEO

4.1 Category Hierarchy

The categories table supports unlimited nesting through the parent_id field:

  • Parent categories have parent_id = NULL
  • Subcategories reference their parent via parent_id
  • Self-referencing foreign key ensures data integrity

4.2 Product Images

Multiple images per product with:

  • Position ordering
  • Primary image designation
  • Cascade delete when product is removed

5. Order Management

5.1 Orders Table

Order Status Flow

pending ‚Üí processing ‚Üí shipped ‚Üí delivered ‚Üí completed

cancelled - Can occur at any stage

Field Type Description
number VARCHAR(255) Unique order number (e.g., ORD-2025-0001)
status VARCHAR(255) Order status (pending, processing, completed, cancelled)
payment_status VARCHAR(255) Payment status (unpaid, paid, refunded)
shipping_status VARCHAR(255) Shipping status (unshipped, shipped, delivered)
grand_total DECIMAL(10,2) Final order total including all charges

5.2 Order Items

Each order contains one or more order items:

  • Stores product snapshot at time of purchase
  • Includes product name, SKU, quantity, and prices
  • Allows product deletion without losing order history

5.3 Shopping Cart

The cart system supports both:

  • Authenticated users: Cart linked via user_id
  • Guest users: Cart linked via session_id
  • Coupon application
  • Automatic total calculations

9. Table Relationships

9.1 Core Relationships

Parent Table Child Table Relationship
users orders One-to-Many (user can have many orders)
orders order_items One-to-Many (order contains many items)

9.2 Foreign Key Constraints

Note: All foreign keys are properly configured with:

  • ON DELETE CASCADE - Child records deleted when parent is deleted
  • ON DELETE SET NULL - Foreign key set to NULL when parent is deleted
